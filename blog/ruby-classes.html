Ruby Classes

Exercise 6.9

March 18, 2016

This week's blog post is on the wonderful ruby object: the Class. You may not know it but your whirring and complex ruby program is like a well oiled engine. That engine is a Class and the block, the alternator, the power steering, and all those components are all methods. Method are defined building blocks that serve a distinct purpose in the larger function, they spray gas in a flammable mist and ingite it, or spin a dynamo to charge up a battery, or filter oil as it flows, but they are just parts of the whole operation, the bigger picture, the Class. To all work in concert they have to have standardized inputs like instance variable. Just as an engine requires it's pulleys to line up and burns a specific type of fuel a methods are set up expecting a specific input and made to proces and send specific types of information.

To define a class you need to title it with a capitalized and descriptive word and end it so it runs like you would a method: 

class Engine

end


You follow that with any attributes you want to set. These are the read/write permissions for the class followed by a symbol you are changing the permissions for. They can be attr_read, attr_write, or attr_accessor (to read and write). 

class Engine
attr_accessor

end


Next you need to initialize the class and you can set any instance variables. Again, instance variable are marked with an @ sign and are recognized by all methods within the class. If you are setting write or accessor attributes for the class this is a good time to make sure those symbols also correspond with instance variables so they can be recognized throughout the class. To initialize the class you write an initialize method like this (make sure you end it like a method too): 

class Engine

         attr_accessor


         def initialize

         @variable = variable

         end

         
end


Now that your class is set up you can get to the business end of things and define the methods you've no doubt described in your pseudocode and preplanning. Those methods fit in after your initialize method and you need to make sure you still end your class after. As your work in these methods remember that any variable that needs to be recognized in multiple methods must be defined as an instance variable with a @ in front of it and depending on your attributes you may be able to read or write some parts from outside the class. 

class Engine
attr_accessor


        def initialize
 @variable = variable
end

def method_one
@longitude = @variable.length
end

def method_two
if @longitude >= 5
puts "longer than 5, dude"
end

end


Now that your Class is chock full of useful methods, attributes, and instance variables you can call it and it's methods with lines like these:

        start_up = Engine.new

        start_up.method_one

        start_up.method_two

        start_up.variable = "I can change my variable from here because of attr_accessor"
