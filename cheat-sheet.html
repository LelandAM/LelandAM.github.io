Cheat Sheet: JavaScript Prototypes

JavaScript is a prototype based language and so fully embracing and understanding these will unlock doors to elegant and flexible coding. In JavaScript every object has a prototype which is itself an object and it is blank by default. The prototype property is what gives objects their properties and methods. When you made an object with object literal it's properties are inherited from Object.prototype. When JavaScript is reading your code and reaches an object it will attempt to find a property in that object and then check to see what is assigned in the prototype.

The prototype property comes in handy for inheritance, it adds methods and properties to an object to make those available to instances of that object. Constructor functions are the most used way to construct prototype chains. Think of it this way, you can easily add a property or method to an object with object literal but to add a new property to a prototype you must use the constructor function: 

function person(first, last, age, eyecolor){
          this.firstName = first;
          this.lastName = last;
          this.age = age;
          this.eyeColor = eyecolor;
        }

Now you have a template for other persons and you can add properties with one line: 

        this.prototype.nationality = "American"

You can also add a new method to an existing prototype: 

person.prototype.name = function(){
            return this.firstName + " " + this.lastName;
            };
